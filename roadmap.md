# 🚀 Passmate (AI 자소서 진단) 개발 로드맵

## 📌 프로젝트 목표
자소서 분석 후 '한시 멘토' 페르소나를 담은 고퀄리티 리포트 자동 생성 및 유료화 런칭

---

## 🎯 Phase 1: Stable MVP & 코어 엔진 (이번 주 최우선 목표)
*목표: 유저 입력 -> AI 분석(에러 방어) -> 결과 확인까지의 뼈대 완성*

- [ ] **1. UI 및 로직 다듬기:** 스플릿 뷰 개편 및 자소서 입력 페이지 다듬기 (기업명, 직무 추가)
- [ ] **2. Gemini API 연동:** 구글 스튜디오 결제 연동 및 1.5 Flash 모델 통신 연결
- [ ] **3. 시스템 프롬프트 최종화:** '따뜻한 팩폭 사수' 페르소나 및 기업 맞춤형 지시어 적용
- [ ] **4. 7대 핵심 예외 처리 (방어 로직 튼튼하게!):**
  - [ ] 최소(200자)/최대(6,000자) 글자 수 제한 및 1,000자 이하 경고 모달창
  - [ ] '분석하기' 버튼 따닥(중복 요청) 방지 (Disabled 처리)
  - [ ] 딴소리 방어, 네트워크 에러 팝업, 로딩 지연(10초 이상) 안내 문구 변경 등

## 🎨 Phase 2: 데이터 저장 & 유저 경험 (Retention)
*목표: 유저가 기록을 보관하고 재방문할 수 있는 환경 구축*

- [ ] **5. 데이터베이스(DB) 구축:** 수파베이스(Supabase) 연동 (유저, 질문, 원문, 리포트 저장)
- [ ] **6. 로그인 연동:** 카카오/구글 소셜 로그인 적용
- [ ] **7. '마이(My)' 탭 구현:** 과거 분석 히스토리 리스트업 및 불러오기 기능
- [ ] **8. 로컬 스토리지 유지:** 새로고침 시 분석 결과 데이터 증발 방지

## ✨ Phase 3: 수익화 로직 & 런칭 준비 (운영)
*목표: 유저 결제 동선 완성 및 트래픽 분석 세팅*

- [ ] **9. 결제 연동 1차:** 토스페이먼츠 샌드박스(테스트) 연동 및 결제 후 리다이렉트 로직
- [ ] **10. 서비스 지표 세팅:** 구글 애널리틱스(GA4) 스크립트 추가
- [ ] **11. 어드민 페이지 구축:** 오늘 조회수, 유저/리포트 현황 등 당장 확인할 수 있는 대시보드
- [ ] **12. 랜딩페이지 마감:** 후킹 문구 및 'Beta', 'Coming Soon' 배너 등 UI 디테일 폴리싱
