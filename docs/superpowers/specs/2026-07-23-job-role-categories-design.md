# 지원 직무 카테고리 설계

## 목표

자소서 분석 화면의 기본 지원 직무 선택지를 폭넓게 제공한다. 사용자는 기획, 마케팅, 경영지원 등 자신의 직무를 빠르게 찾고 선택할 수 있어야 하며, 목록에 없는 직무는 기존처럼 직접 입력할 수 있어야 한다.

## 화면 구성

- 기존의 둥근 직무 선택 버튼과 단일 선택 동작을 유지한다.
- 직무 버튼을 다음 카테고리별 섹션으로 나눈다: 기획·PM, 마케팅·브랜딩, 경영·사업, 재무·회계, 구매·SCM, 인사·총무, 영업·고객, 데이터·IT, 디자인·콘텐츠.
- 모든 카테고리는 기본으로 펼쳐 표시한다. 별도 아코디언, 검색 단계, 선택 단계는 추가하지 않는다.
- 기존 `서비스 PM` 프리셋은 `서비스 기획`으로 교체한다.
- `+ 직접 입력`은 모든 카테고리 아래에 한 번만 둔다.

## 직무 목록

- 기획·PM: 서비스 기획, 제품/상품 기획, 사업 기획, UX 기획, 프로젝트 매니저
- 마케팅·브랜딩: 브랜드 마케팅, 디지털 마케팅, 퍼포먼스 마케팅, 콘텐츠 마케팅, CRM 마케팅
- 경영·사업: 경영전략, 사업전략, 신사업, 사업개발, 해외사업
- 재무·회계: 재무, 회계, 세무, IR, 자금, FP&A
- 구매·SCM: 구매, 전략구매, SCM, 물류, 생산관리
- 인사·총무: 인사(HR), 채용, 조직문화, 교육, 총무
- 영업·고객: 국내영업, 해외영업, B2B 영업, 고객관리, 고객지원(CS)
- 데이터·IT: 데이터 분석, 데이터 사이언스, 프론트엔드 개발, 백엔드 개발, AI/ML 엔지니어
- 디자인·콘텐츠: UX/UI 디자인, 프로덕트 디자인, 그래픽 디자인, 콘텐츠 기획, 영상 콘텐츠

## 구현 경계

`Analyze.tsx`의 단일 문자열 배열을 카테고리와 직무 배열을 가진 상수로 바꾼다. 선택 상태와 분석 요청에 전달하는 직무 문자열의 형식은 변경하지 않는다. 따라서 저장, 리포트, API에는 영향을 주지 않는다.

## 검증

- 카테고리 데이터에 요청된 핵심 직무(서비스 기획, 제품/상품 기획, 브랜드 마케팅, 경영전략, 재무, 구매)가 포함되는지 테스트한다.
- 기존 선택/해제 및 직접 입력 경로는 변경하지 않는다.
- 타입 검사와 프로덕션 빌드로 렌더링 및 번들 호환성을 검증한다.
