# 실무자 코멘트 댓글 스레드 설계

## 목표

하단 `실무자 코멘트`를 하나의 긴 AI 리포트처럼 보이는 블록에서, 한 명의 현직 멘토가 시간순으로 남긴 세 개의 댓글처럼 바꾼다. 기존 다크 테마와 절제된 프리미엄 분위기는 유지한다.

## 범위

- `ReportResult`의 제일 아래 `section-pm-comment`만 변경한다.
- 상단 요약 카드 안의 `현직자 코멘트`는 변경하지 않는다.
- 기존 `pmComment` 문자열과 세 블록으로 분리하는 `splitMentorComment` 데이터 흐름은 유지한다.
- 표시 제목은 순서대로 `읽힌 인상`, `더 선명해질 지점`, `면접에서 준비할 것`을 사용한다.

## 화면 구조

각 블록은 독립된 댓글 항목이며, 다음 순서만 포함한다.

1. 원형 아바타
2. `Mentor Hansi`와 `방금 전`
3. 댓글 제목
4. 댓글 본문

댓글을 감싸는 카드 배경, 그림자, 굵은 테두리는 사용하지 않는다. 왼쪽 아바타 사이에는 낮은 대비의 1px 세로 연결선을 두어 하나의 댓글 스레드로 읽히게 한다. 댓글 사이에는 충분한 세로 여백을 둔다.

## 타이포그래피와 색상

- 멘토 이름과 제목은 본문보다 한 단계 높은 대비와 굵기로 표시한다.
- 제목은 본문보다 약간 크게 보여 스캔이 가능해야 한다.
- 작성 시각과 본문은 보조 텍스트 대비를 사용한다.
- `01`, `02`, `03` 번호, 색상별 제목, 굵은 컬러 라인, 강조용 좌측 막대는 제거한다.
- 강조 마크업이 포함된 본문도 기존의 안전한 텍스트 렌더링을 거쳐 표시한다.

## 반응형 동작

- 데스크톱과 모바일 모두 아바타 열과 본문 열을 유지한다.
- 좁은 화면에서는 아바타와 세로 연결선의 크기·위치를 줄이되, 본문 폭은 줄바꿈으로 확보한다.
- 댓글 사이의 순서와 메타데이터는 모든 화면 크기에서 유지된다.

## 검증

- 화면 구조 테스트는 하단 코멘트 섹션에 세 번의 멘토 이름과 작성 시각, 요청한 세 제목, 아바타와 스레드 연결 표현이 있는지 확인한다.
- 같은 테스트에서 번호 스타일, 색상별 제목 스타일, 컬러 막대가 더 이상 렌더링 소스에 없는지 확인한다.
- 관련 Vitest 테스트와 TypeScript 검사를 실행한다.

## 제외 범위

- 상단 `현직자 코멘트` 요약의 레이아웃 변경
- API, 프롬프트, 데이터베이스 스키마, `pmComment` 저장 형식 변경
- 멘토별 프로필이나 실제 댓글 작성 기능 추가
