# 프리미엄 분석 이용권 및 그로블 결제 연동 설계

## 목적

PassMate의 분석 서비스를 계정 단위 이용권으로 운영한다. 프리미엄 판매가 비활성화된 동안에는 계정당 무료 분석 1회만 제공한다. 관리자가 프리미엄 판매를 활성화하면 랜딩과 분석 화면에 프리미엄 3회 이용권을 노출하고, 그로블 일반결제 완료 웹훅을 통해 구매 계정에 이용권을 자동 지급한다.

## 제공 정책

| 상태 | 제공 분석 횟수 | 노출 |
| --- | --- | --- |
| 프리미엄 판매 OFF | 계정당 무료 1회 | 무료 플랜만 노출 |
| 프리미엄 판매 ON | 무료 1회 + 구매당 프리미엄 3회 | 무료와 프리미엄 플랜 함께 노출 |

- 분석은 결과가 정상 저장될 때 1회 차감한다. 실패하거나 취소된 분석에는 차감하지 않는다.
- 프리미엄 구매는 누적된다. 예를 들어 두 번 결제하면 프리미엄 분석 6회가 추가된다.
- 무료 1회는 별도 소진 여부로 관리하고, 프리미엄 잔여 횟수와 합산하여 사용자에게 표시한다.

## 데이터 모델

Supabase에 다음 세 테이블 또는 동등한 데이터 구조를 추가한다.

### `entitlement_settings`

단일 설정 레코드다.

- `premium_enabled`: 프리미엄 노출과 구매 흐름 활성화 여부
- `premium_credits_per_purchase`: 기본값 3
- `groble_payment_url`: 현재 상품 결제 링크 `https://www.groble.im/payment/4SGBV5`

### `analysis_entitlements`

사용자별 집계다.

- `user_id`
- `free_credits_granted` (기본 1)
- `free_credits_used` (기본 0)
- `premium_credits_granted` (기본 0)
- `premium_credits_used` (기본 0)

남은 횟수는 지급 횟수에서 사용 횟수를 뺀 값으로 계산한다. 클라이언트가 아닌 서버가 이 값을 검증하고 변경한다.

### `payment_entitlements`

그로블 결제 이벤트와 지급 이력을 보관한다.

- `provider` (`groble`)
- `provider_payment_id` (유니크)
- `user_id`
- `status` (`paid`, `cancel_requested`, `refunded` 등)
- `credits_granted`
- `raw_event` (검증/운영 목적)

`provider_payment_id` 유니크 제약으로 웹훅 재전송이 와도 이용권은 한 번만 지급한다.

## 결제와 계정 연결

결제 시작 전 로그인 사용자의 식별자를 결제 주문 메타데이터 또는 결제 링크에 전달할 수 있는 경우 반드시 함께 전달한다. 그로블이 이를 지원하지 않는 경우에는 다음의 보조 흐름을 사용한다.

1. 사용자가 프리미엄 선택을 누르면 서버가 `pending purchase`를 만든다.
2. 해당 구매의 안전한 토큰을 결제 전환/복귀 URL에 포함한다.
3. 그로블 일반결제 완료 웹훅이 결제 ID와 주문 식별자를 보내면 서버가 보류 구매와 연결한다.
4. 연결이 불가능한 이벤트는 자동 지급하지 않고 관리자 결제 관리 화면에서 검토한다.

자동 지급은 결제 성공 웹훅만 신뢰한다. 사용자가 결제 완료 화면으로 복귀한 사실만으로는 이용권을 지급하지 않는다.

## 웹훅

그로블 연동 화면의 웹훅 URL에는 배포된 API 엔드포인트를 등록한다.

`https://passmate-gamma.vercel.app/api/webhooks/groble`

수신 이벤트는 `일반결제 완료`만 활성화한다. 취소/환불의 정확한 이벤트 페이로드와 인증 방식은 그로블 가이드에서 확인한 뒤 후속 단계로 추가한다.

웹훅 처리기는 다음을 수행한다.

1. 요청 인증 정보와 결제 완료 상태를 검증한다.
2. 결제 ID가 이미 처리되었는지 확인한다.
3. 결제와 사용자를 연결한다.
4. `payment_entitlements`를 기록하고 사용자에게 프리미엄 3회를 원자적으로 추가한다.
5. 성공 응답을 빠르게 반환한다.

## 화면 구성

### 랜딩

- Premium OFF: 무료 분석 1회 플랜만 보여 준다.
- Premium ON: 무료 분석 1회와 프리미엄 분석 3회 플랜을 함께 보여 준다.
- 프리미엄 CTA는 그로블 결제 링크로 이동한다. 로그인하지 않은 사용자는 먼저 로그인한 뒤 결제로 이어진다.

### 분석 화면

- 상단 또는 분석 실행 영역에 `남은 분석 n회`를 표시한다.
- 잔여 횟수가 0이면 분석 실행을 막고, Premium ON일 때만 프리미엄 구매 CTA를 표시한다.
- 서버의 이용권 확인 결과를 기준으로 표시한다. 브라우저 저장소 값만으로 접근을 허용하지 않는다.

### 마이페이지

- 현재 상태(무료 이용 중 또는 프리미엄 이용권 보유)를 표시한다.
- 무료/프리미엄별 사용 횟수와 합산 잔여 분석 횟수를 표시한다.
- Premium ON이고 잔여 횟수가 0이면 구매 CTA를 표시한다.

### 관리자

- Settings에 `프리미엄 판매 활성화` 토글을 추가한다.
- 결제 관리 화면에서 결제 ID, 사용자, 지급 횟수, 상태, 생성 시각을 확인한다.
- 자동 연결에 실패한 결제만 수동으로 사용자에게 연결/지급할 수 있다. 임의의 중복 지급은 금지한다.

## 오류 처리와 보안

- 이용권 확인, 차감, 지급은 모두 서버 API에서 수행한다.
- 웹훅 서명/공유 비밀이 제공되면 반드시 검증한다. 제공되지 않으면 공급자 결제 조회 API로 결제 상태를 교차 검증하는 방식을 우선 검토한다.
- 웹훅의 필수 식별자가 없거나 사용자를 연결할 수 없으면 2xx 성공으로 위장하지 않고 운영자가 확인 가능한 오류 상태로 기록한다.
- 분석 저장 실패 시 차감 트랜잭션을 롤백한다.

## 검증

- Premium OFF에서 랜딩은 무료 플랜만 보여 준다.
- Premium ON에서 두 플랜과 그로블 결제 CTA가 보인다.
- 무료 계정은 1회 분석 후 다시 실행할 수 없다.
- 정상 웹훅 1회는 프리미엄 3회를 지급하고, 같은 결제 이벤트의 재전송은 중복 지급하지 않는다.
- 분석 결과가 저장될 때만 잔여 횟수가 줄어든다.
- 마이페이지, 분석 화면, 서버 응답의 남은 횟수가 일치한다.
